## 🏗️ 关键实现

### 1. Agent Core — 情感时序修复与 L3 Prompt 注入
- **文件**: `chronopersona/agent_core/state_machine.py` (+45/-12)
- **决策**: 将 `self._emotion_state = _update_emotion(user_input)` 从 `run_turn()` 末尾迁移到 `ActionPlanner.plan()` 调用之前，确保调制参数基于当前轮次情感状态而非上一轮滞后值；`_build_prompt()` 追加 `[Emotion State]`、`[Semantic Facts]`、`[Insights]` 三个文本段，使 LLM Node 首次获得 L3 语义事实与主动反思洞察。
- **影响面**: ActionPlanner 调制参数正确性（A10 通过）、LLM 上下文完整性（A8 通过）、A1-A11 端到端评估无回归。
- **关联设计文档**: `requirements.md` 4.9（Emotion Engine）、4.6（混合召回融合）

### 2. FaissEpisodicStore — 删除一致性修复
- **文件**: `chronopersona/memory_system/l2_episodic/faiss_store.py` (+18/-3)
- **决策**: `IndexFlatIP` 不支持物理删除，原 `delete()` 仅清空元数据 `content` 导致向量仍被检索命中，返回 `content=''` 的幽灵记忆。引入 `_deleted_indices: Dict[str, Set[int]]` 按 branch 追踪已删 FAISS 下标，在 `retrieve()` 中过滤；同时防御性跳过 `entry is None` 的反查。
- **影响面**: L2 存储层删除语义正确，`test_faiss_store.py::test_delete_removes_from_index` 从失败转为通过。
- **关联设计文档**: `requirements.md` 4.4（L2 Episodic Memory）

### 3. GridWorldAdapter — 跨本体真实动作映射
- **文件**: `chronopersona/embodied/grid_world_adapter.py` (+85/-12)
- **决策**: 将 5 个高层 `action_token`（`approach_gently` / `retreat_slowly` / `turn_to_user` / `interact` / `look_around`）从 stub `"mock_{token}"` 改为真实跨本体映射：`grid_2d` 映射为 `move_forward` / `move_backward` / `turn_toward` / `interact_with` / `scan_fov`；`ros2_mobile` 映射为 `cmd_vel` / `navigate_to` 等，参数按 `speed_mult` 调制。
- **影响面**: A9 跨本体迁移评估通过（`robot_type` 区分验证），W7 具身智能核心交付物就位。
- **关联设计文档**: `requirements.md` 4.10（2D Virtual Environment）

### 4. A1-A11 对抗评估基线与自动化 Runner
- **文件**: `tests/test_a1_a2_a3.py` ~ `test_a11_persona_drift.py`（14 个新增文件），`evaluation/runner.py`（重构）
- **决策**: 覆盖记忆召回、角色隔离、多端冲突、意图图谱导航、情感一致性、具身感知、跨本体迁移、动作可审计、人格漂移检测 9 大场景；`runner.py` 从 subprocess 调用 pytest 改为直接执行业务逻辑产出 JSON 量化报告，避免重复执行与日志吞掉。
- **影响面**: W6 核心交付达成，`make eval` 6/6 PASS；为 W8 面试量化成果提供唯一可信数据源。
- **关联设计文档**: `requirements.md` 8.2（评估框架）

### 5. 全仓类型注解与 `branch_id` 契约硬化
- **文件**: 全仓 20+ 文件（`sliding_window.py`, `intent_graph.py`, `state_machine.py`, `faiss_store.py`, `mock_*.py` 等）
- **决策**: 统一替换 Python 3.10+ 内置泛型语法（`list[...]` / `dict[...]` / `X | Y`）为 `typing` 模块（`List` / `Dict` / `Optional`）；所有公共数据操作方法显式声明 `branch_id: str` 并增加 `if not branch_id: raise ValueError` 防御。
- **影响面**: 消除 24 处 `TypeError`（测试调用方未同步参数导致），符合 AIDER.md 第 2/3 条接口契约；兼容 Python 3.10/3.11。
- **关联设计文档**: `AIDER.md` 第 2 条（接口契约）
